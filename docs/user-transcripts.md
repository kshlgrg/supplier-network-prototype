# User Interview Transcripts: Vypaar Saathi

These short transcripts summarize four prototype walkthrough interviews. They are written as concise research evidence for the design process document.

## Interview 1: Low Digital Confidence User

**Profile:** Small business operator who mainly uses phone calls and WhatsApp for supplier updates.

**Interviewer:** What did you understand from the first screen?

**User:** It is telling me to create a workspace and then add supplier details. That is clear.

**Interviewer:** Was the sign-up difficult?

**User:** No. One question at a time was easy. I did not feel like I was filling a long form.

**Interviewer:** What helped most?

**User:** The checklist. It showed what was done and what was left.

**Interviewer:** What should improve?

**User:** Use simple words everywhere. Some analytics words may need small explanations.

## Interview 2: Medium Digital User

**Profile:** MSME operations user who maintains supplier information in spreadsheets.

**Interviewer:** How does this compare with your current spreadsheet?

**User:** The app is easier because it shows the result after entering data. In a spreadsheet, I need to calculate and compare myself.

**Interviewer:** Which screen felt most useful?

**User:** Pick Best. It showed why one supplier is better based on weights.

**Interviewer:** Did you trust the recommendation?

**User:** I trusted it more when I could see the reason and change the weights.

**Interviewer:** What should improve?

**User:** Add Excel upload later, because typing all suppliers manually will take time.

## Interview 3: High Digital User

**Profile:** Procurement user familiar with dashboards and vendor scorecards.

**Interviewer:** What did you think of the risk and simulation screen?

**User:** It makes the prototype feel more professional. Showing uncertainty is better than showing only one fixed score.

**Interviewer:** Was anything confusing?

**User:** The model should always show what assumptions it is using.

**Interviewer:** What is the strongest part?

**User:** The flow from data entry to dashboard, comparison, simulation, and best plan feels connected.

**Interviewer:** What should improve?

**User:** Add historical supplier performance data so the simulation becomes more realistic.

## Interview 4: Digitally Advanced User

**Profile:** User comfortable with analytics tools, scenario planning, and operational dashboards.

**Interviewer:** Would this be useful for supplier planning?

**User:** Yes, as a lightweight decision support tool. It is not an ERP, but it explains the decision journey well.

**Interviewer:** What did you notice about the interface?

**User:** The simple labels are good. They make the tool usable for non-technical people.

**Interviewer:** Did the best-plan screen help?

**User:** Yes. A recommendation is useful only when it gives reasons and constraints.

**Interviewer:** What should improve?

**User:** Add saved scenarios and exportable reports for management review.

## Key Takeaways

- Users preferred one guided sign-up path instead of multiple setup options.
- Users wanted every data field to create a visible output.
- Trust increased when recommendations showed reasons, weights, and assumptions.
- Future versions should support spreadsheet upload, saved scenarios, and report export.
